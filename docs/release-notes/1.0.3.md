# Expeditions Macro 1.0.3

This patch makes camera alignment recover from an incorrect fast yaw estimate, blocks placement after an unresolved alignment, and standardizes model recording across Windows profiles.

## Changes

- A fast alignment result below the camera model target now triggers a measured full-turn scan.
- The fallback scores stable captures throughout the turn, returns to the strongest candidate, and applies micro-drag refinement.
- Unit placement cannot start unless the final camera score meets the model confidence target.
- Camera calibration and placement recording temporarily use the detector pack's 808 by 611 Roblox client size and restore the original window afterward.
- Expeditions startup verifies that Roblox accepted the required client size before automation continues.
- Camera and placement model descriptions now sit below their section headings and above wider controls instead of occupying a permanent side column.
