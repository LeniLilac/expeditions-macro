# Expeditions Macro v1.3.0-beta.11

This silent beta restores beta.8 camera-calibration stability while keeping the Windows 10 capture repair introduced in beta.9, and hardens unattended runs against inactivity recovery. It does not send a Discord release announcement.

## Camera capture freshness

- Camera setup, manual Auto Align, and macro alignment now discard queued compositor frames before scoring movement.
- The scorer receives the newest available Roblox frame instead of a pose approximately two input steps behind.
- This fixes camera setup that rejected a reversible fine sweep in beta.10 or continued scanning indefinitely in beta.9.
- The Windows 10-safe notification-only capture callback and HDR-aware window capture remain intact.

## Long-idle reliability

- Expedition, Challenge, Story, Raid, and Infinite gameplay sends one ordinary `O` pulse every eight minutes.
- Challenge cooldown waits use the same keepalive.
- A transient focus or input failure retries after one minute without stopping the task.
- After the AFK Chamber's Return to Lobby action, Story and Raid wait through teleport/loading before testing the configured Play key.

## Validation

- Added a queued-frame regression proving stale compositor frames are disposed and only the newest frame is scored.
- Added keepalive timing, retry, and AFK handoff policy regressions.
- The complete 531-test application and detector corpus plus all 5 Deep Debug Viewer tests pass.
- Repository policy, release metadata, checksums, native dependency verification, and a 22-page packaged-app snapshot smoke test pass.
