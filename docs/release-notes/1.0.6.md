# Expeditions Macro 1.0.6

This patch hardens Expedition UI recognition and clicking against the small position and scale differences Roblox can render across Windows profiles and clients.

## Changes

- Continue and checkpoint pauses are recognized from their visible green and red button geometry, including the shifted Continue layout reported after v1.0.5.
- Start, pause, recovery, map, difficulty, confirmation, extraction, reward, victory, and defeat actions now derive their click positions from the current Roblox frame instead of assuming one fixed coordinate.
- Lobby and Play recovery, map and difficulty selection, the node progress bar, and the unit hotbar can follow translated or scaled UI regions while Roblox remains at the required 808 by 611 client size.
- Reward recognition uses the three stable mouse icons beneath the cards and remains independent of card rarity and color.
- Victory and Defeat recognition combines their distinct title banner with the Repeat Stage control to avoid relying on the changing scene behind the UI.
- Regression coverage now verifies all 197 captured UI images classify as their intended state without cross-state false positives, plus translated, scaled, and combined layout variants for every macro state and action.
- The reported Continue frame is included in the public detector dataset.

Existing camera and placement models remain compatible and do not need to be recreated.
