# Expeditions Macro v1.2.0

## Highlights

- Camera setup no longer needs a hand-drawn comparison region. It standardizes zoom and top-down pitch, then chooses four stable map regions across the Roblox client automatically.
- Coarse yaw now uses sensitivity-independent Left/Right-arrow pulses. Fine mouse refinement learns a signed neighborhood around the goal and saves that atlas for later alignments.
- Runtime alignment gets three fresh attempts with alternating scan directions and sampling phases. It combines the saved coarse and fine atlases with a full-turn fallback instead of accepting a low-confidence view.
- If every alignment attempt misses its confidence target, no units are placed and the match is not started. The macro exits safely and reports the skipped task; Challenge mode continues to its next configured task while standalone Expeditions stops at the party preview.
- Settings now includes a click-through UI-scale calibration overlay, and Discord reporting includes a **Test webhook** action.
- Cursor parking uses spaced Roblox-acknowledged motion pulses so hover cards and highlighted selector rows clear before detection and clicks.

## Setup note

v1.2.0 uses camera-model schema version 3. Existing camera models from v1.1.x must be recorded again. Existing placement models and presets remain usable after selecting the replacement camera models.

Detector pack 1.0.2 remains bundled and upgrades or repairs older installed packs automatically.
