# Expeditions Macro 1.0.9

This patch fixes two cross-system failures reported after v1.0.8: lobby recovery could miss the Expeditions mode tile when its map content differed, and camera-model setup could scan past a real but coarsely sampled full-turn return.

## Changes

- Play-screen recovery now detects two stable parts of the Expedition tile: its teal title treatment and its dark progress/action footer. Map names, artwork, reward icons, and avatars are no longer part of the decision.
- The Expeditions click is mapped through the same detected scale and translation, so the app clicks the current tile location instead of a stale fixed coordinate.
- A reported cross-user Play screen with a different current map, avatar, icons, and Roblox UI layout is included in the public golden-image corpus.
- Camera setup now treats a prominent 70-75% coarse local peak as provisional. It checks that the next yaw sample repeats the beginning of the scan, then performs the existing one-pixel fine sweep around the candidate.
- A provisional camera candidate is accepted only when fine refinement recovers to the existing strong confidence level. An isolated lookalike without the matching continuation is still rejected.
- If the scan reaches its sample limit, setup revisits the best continuation-verified candidate once before returning to the start and reporting failure.
- Added positive and negative camera regressions modeled on the reported 78% wraparound, plus full cross-state Play/lobby regression coverage.
- GitHub Releases now announce automatically in the public Discord release channel using Components V2, and the repository includes an `AGENTS.md` contributor guide.

Existing camera models, placement models, presets, and detector pack 1.0.1 remain compatible. The detector-pack version is unchanged because the Play fix is implemented by the app's specialized detector.
