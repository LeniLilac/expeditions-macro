# Expeditions Macro 1.0.14

This release fixes automatic recovery getting stuck at the initial in-map Continue screen after teleporting from the lobby and updates the community Discord invite.

## Changes

- Recovery now recognizes the standalone green **Continue** button that can appear after the teleport preview, clicks its detected Roblox-relative center, and proceeds to the Start Game dialog.
- The new transition is accepted only while leaving the teleport preview. Map preview remains authoritative while it is still visible, and ordinary in-run Continue/checkpoint handling is unchanged.
- Updated the **Join Discord** sidebar button and README to use `https://discord.gg/wE6XSVyXsN`.
- Added recovery-policy regressions for the new transition, its guarded scope, and map-preview priority.

The supplied v1.0.12 diagnostic capture reproduced at 100% Continue confidence with an action at `(404, 490)`. Existing camera models, placement models, presets, and detector pack 1.0.1 remain compatible. The detector-pack version is unchanged because the existing detector already recognizes this screen; the defect was in recovery routing.

## Downloads

- `ExpeditionsMacro-1.0.14-win-x64-setup.exe` - recommended Windows installer.
