# Expeditions Macro 1.0.12

This patch prevents delayed Roblox UI transitions from causing repeated checkpoint extraction clicks.

## Changes

- Checkpoint extraction now runs as one guarded transaction: click Extract once, wait for the confirmation, click Confirm once, and wait for the confirmation dialog to close before returning to gameplay monitoring.
- The macro no longer returns to generic checkpoint handling after a six-second confirmation miss, which previously allowed another Extract click during server or client lag.
- Both extraction transition windows now allow up to 30 seconds. If Roblox never acknowledges the action, the macro stops with an actionable message instead of sending another blind click.
- An extraction confirmation already visible when monitoring resumes receives the same one-click-and-wait treatment.
- Added regression coverage proving repeated screen observations cannot authorize duplicate Extract or Confirm actions.

Existing camera models, placement models, presets, and detector pack 1.0.1 remain compatible. The detector-pack version is unchanged because this fix is enforced by the app state machine.
