# Expeditions Macro 1.1.0

This release adds complete regular-Challenge rotation alongside the existing Expeditions workflow.

## Challenge mode

- Added a dedicated **Challenges** workspace for Trait, Stat, and Sprite Challenges, which reset globally at `xx:00` and `xx:30`.
- Recognizes School Grounds, Flower Forest, Rose Kingdom, Fairy King Forest, and King's Tomb, then loads each map's configured camera and placement models.
- Supports separate before-start and delayed after-start placement phases.
- Adds configurable defeat retries, with zero retries by default and eligibility restored at the next global reset.
- Can wait during cooldown or run a selected Expeditions preset until the next Challenge window.
- Infers the daily limit only when all three entries remain unavailable across a complete global reset, then waits until midnight UTC.
- Sends optional Components V2 Challenge status, attempt, recovery, Victory, and Defeat reports through the existing protected Discord webhook setting.

## Reliability

- Uses state-aware transitions from lobby through selector, preview, prestart, gameplay, terminal screens, and Change Gamemode.
- Handles both observed Challenge panel scales, private-party previews, dimmed selector rows, cooldown-only selectors, animated thumbnails, reward tooltips, and hovered controls.
- Clicks Challenge entries through stable map artwork instead of reward icons.
- Recognizes bright game-mode artwork from stable headings and divider geometry instead of requiring dark tile content.
- Includes the v1.0.14 and v1.0.15 post-teleport and Continue-confirmation reliability fixes in the Challenge-capable build.

## Validation

- Passed 219 automated tests across unit, state-machine, action-placement, and golden-image coverage.
- Regression-tested 336 checked-in Roblox client captures: 269 Expeditions fixtures and 67 Challenge fixtures.
- Existing camera models, placement models, Expeditions presets, and detector pack 1.0.1 remain compatible.
