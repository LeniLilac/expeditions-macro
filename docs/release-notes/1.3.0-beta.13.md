# Expeditions Macro v1.3.0-beta.13

This silent beta fixes a transient Windows capture failure observed while Roblox changes from a launch-ready party into a stage teleport. It does not send a Discord release announcement.

## Stage-launch reliability

- If Windows Graphics Capture pauses beyond the normal one-second fresh-frame deadline during a Roblox compositor handoff, the macro disposes that stalled capture session and creates one bounded replacement.
- The replacement uses the existing longer initial-frame allowance and resumes detector polling from its first fresh frame.
- The macro does not repeat **Start**, another navigation click, or camera input while rebuilding capture.
- A second timeout still stops safely and remains eligible for configured outer recovery.
- Cached pre-teleport frames remain forbidden, preserving the post-input freshness guarantee introduced in beta.12.

## Validation

- Added focused regressions proving a transient fresh-frame timeout recreates one session and a repeated timeout remains bounded.
- Reproduced the failure from a beta.12 Spirit City Raid 2 Deep Debug archive containing the exact one-second post-barrier timeout while the same Roblox PID, window, and canonical client geometry remained valid.
- The complete 534-test application/detector suite and all 5 Deep Debug Viewer tests pass.
- Release build, repository policy, scoped formatting, and diff checks pass.
