# Expeditions Macro v1.2.1

## Highlights

- Expedition map selection now verifies the stable cyan active-row marker instead of localized names or changing preview artwork. This fixes false “Map 1/2 could not be selected” errors across current, alternate, and French layouts.
- Camera preparation is more tolerant of real client differences: it clamps zoom and pitch, evaluates both shift-lock states, tolerates small translation and scale shifts, weakly uses hue to resolve close matches, and verifies the result over three fresh frames.
- Zoom-out uses Roblox's `O` key first, with mouse-wheel input retained as a fallback.
- King's Tomb and similar prestart screens now prioritize the centered **Start Game** dialog over post-match HUD lookalikes.
- Extraction confirmation is freshly detected and retried up to three times when Roblox misses a click.
- Roblox discovery now validates the player executable and PID, refreshes stale handles after teleports, and retries focus. When Windows clamps the normal window, a temporary borderless fallback can force the exact 808 by 611 client size used by automation.
- Automatic error diagnostics default to enabled and package the latest ten action-state frames, ten additional frames captured at half-second intervals after failure, and the current run log. Only the ten newest automatic error archives are retained.

## Compatibility

Camera models recorded for v1.2.0 remain compatible. Detector pack 1.0.2 remains bundled and is repaired or upgraded automatically when required.
