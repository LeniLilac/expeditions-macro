# Expeditions Macro v1.3.0-beta.1

This beta expands Expeditions Macro into a prioritized multi-mode task runner and adds opt-in forensic diagnostics for difficult automation failures. It is published silently as a GitHub prerelease for early testing and does not send a Discord release announcement.

## Highlights

- Added saved Macro plans that prioritize Challenge, Expedition, Story, and Raid tasks, preserve progress, and switch modes only between matches.
- Added Story automation for five maps across Acts 1-5, Infinite, and Mastery, including Normal/Hard selection, two placement phases, defeat retries, recovery, and optional saved-team loading.
- Added Raid automation for Spirit City Acts 1-3 with the same placement, retry, recovery, and saved-team controls.
- Added automated Team 1-8 loading through the in-game Units interface and a separately configured Unit-menu key.
- Added a disabled-by-default Deep Debug mode. When explicitly enabled, every successful, canceled, or failed operation produces a potentially multi-gigabyte archive containing detector frames, state/action traces, generated input, sanitized settings, selected plans and presets, the detector pack, and referenced camera and placement models. Protected webhooks and Discord user IDs are excluded.
- Updated the interface with embedded Fredoka typography, native Lucide icons, larger switches, responsive action rails, preset deletion, and dark/light layout fixes.

## Reliability fixes

- Challenge placement coordinates covered by Start Game are deferred until after the deliberate Start click, preventing accidental early starts and swallowed placements.
- Challenge stage teleporting receives an extended prestart wait only after the stable Roblox transition is detected.
- Story, Raid, and Challenge team loading now begins only after prestart is verified.
- Play navigation stops with explicit in-game keybinding instructions when the configured Toggle Play Menu key does not open the interface.
- Camera preparation retries the alternate shift-lock state, and new camera models default to a 200 ms settle interval.
- Safe preset deletion reports any Macro-plan or fallback references that must be removed first.

## Beta note

This is an early public build of the new scheduler, Story, Raid, team-loading, and Deep Debug workflows. Keep Deep Debug disabled during ordinary runs; its lossless frame history can consume multiple gigabytes per operation and is never deleted automatically.
