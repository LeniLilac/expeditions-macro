# Expeditions Macro v1.3.0-beta.10

This silent beta makes long unattended plans recover from failed Roblox sessions and repairs a device-dependent camera calibration loop. It does not send a Discord release announcement.

## Roblox recovery

- Macro plans can optionally store a private-server share link under **Roblox reconnect**.
- Normal lobby, disconnect, AFK, party, and missing-world recovery runs first.
- If bounded recovery still fails, the app closes only the verified Roblox player process, launches the saved server through Windows' registered `roblox://` protocol, reloads plan progress, and retries the same incomplete task.
- No browser, Roblox credentials, cookies, memory access, or process injection is used.
- The link is DPAPI-protected and excluded from logs and diagnostics. A three-restarts-per-ten-minutes circuit breaker prevents relaunch loops.

## Camera reliability

- A reported blue-void prestart state is detected before camera movement or unit placement. The current route is retried through Play instead of consuming camera attempts.
- Fine calibration now uses identical atomic right-drag gestures for forward and reverse movement.
- Setup verifies the actual zero view and the returned zero view before accepting the fine atlas.
- A false fine-neighborhood candidate restores its scan pose and continues the current full turn; it can no longer recursively trigger another 360-degree scan.
- The changes preserve the existing multi-frame scoring and full-resolution acceptance gates rather than trading accuracy for scan speed.

## Navigation fixes

- Map 1 verification accepts a selected row whose changing preview artwork is bright.
- Challenge Back navigation waits for the detail-to-selector transition before another click.
- Expedition startup can leave a retained post-match party before navigating the configured route.

## Validation

- Focused camera, world-readiness, navigation, scheduler, private-link, persistence, and secret-redaction regressions pass.
- The complete detector corpus, Release test suite, repository policy, dark/light UI snapshots, release-package verification, and packaged-app smoke checks are required before tagging.
