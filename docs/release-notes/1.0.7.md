# Expeditions Macro 1.0.7

This patch fixes the long-running Start-pause stall and adds automatic recovery when Roblox moves an inactive player to the AFK Chamber.

## Changes

- Every simulated click now parks the cursor at a neutral Roblox client edge. This prevents hover colors from changing the next instance of Start, Continue, checkpoint, confirmation, recovery, or terminal controls.
- Start Game recognition now accepts the hovered and transition appearances captured in the reported session while retaining the dialog geometry and header checks that prevent unrelated matches.
- AFK Chamber is now a recovery state. The macro clicks **Return to Lobby**, waits for the lobby, and then uses the existing Play, map, difficulty, preview, and prestart flow to rejoin the configured match.
- AFK recovery is counted and can produce the same optional Discord recovery notification as a disconnect or unexpected teleport.
- Regression coverage includes five AFK Chamber frames, four hovered Start frames, UI scale/translation checks, and cross-state false-positive checks.

Existing camera models, placement models, presets, and detector pack 1.0.1 remain compatible.
