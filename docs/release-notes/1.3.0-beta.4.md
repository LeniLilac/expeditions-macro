# Expeditions Macro v1.3.0-beta.4

This silent beta consolidates fixes reported through beta.3 Deep Debug captures. It does not send a Discord release announcement.

## Camera alignment

- Final full-resolution verification now tolerates a coherent cross-session vertical projection shift only when at least three of the four saved world regions independently agree on the translation.
- Thumbnail atlas matching retains its tight five-pixel window, and nearby or wrong yaw remains below the saved success threshold.
- After a complete arrow-key turn and return, the engine captures the live pose again before applying a fine mouse correction. It no longer reuses a stale correction from the first visit to that yaw.
- Strong fine-neighborhood candidates can be tested before a complete fallback turn when session rendering lowers their score below the original setup baseline. The unchanged direct threshold and three-frame verification remain the final gate.

## Mode navigation and results

- Story recognizes the settled upper party-preview rail after its entrance animation and clicks the Start action mapped from the current frame.
- Story and Raid recognize the current compact Victory action rail, so completed matches no longer remain stuck on the results screen.
- A Challenge rotation entering cooldown closes its own panel and verifies the shared game-mode selector before the scheduler starts another mode.
- Recoverable Story/Raid camera failures restore shared game-mode navigation before yielding to the scheduler; an unverifiable handoff stops safely.

## Model tools

- Placement recording and playback snapshot timing switches on the WPF thread before background work starts, preventing cross-thread ownership errors.

## Validation

- Added reviewed King's Tomb Mastery preview and compact Spirit City Victory fixtures.
- Added privacy-safe world-region composites from a reported camera failure and regressions for projection drift, nearby and wrong yaw, and post-turn re-observation.
- Added fail-closed Challenge handoff retries and scheduler ownership coverage.
- The release is built only after the complete local Release suite, all checked-in image regressions, dark/light UI verification, checksums, and installer/portable smoke checks pass.
