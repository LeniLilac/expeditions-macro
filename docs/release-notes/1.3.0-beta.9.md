# Expeditions Macro v1.3.0-beta.9

This silent beta repairs Windows Graphics Capture failures reported on Windows 10. It does not send a Discord release announcement.

## Window capture

- Frame-arrival callbacks no longer access apartment-bound WinRT frame or Direct3D objects, preventing the `RPC_E_WRONG_THREAD` error that could stop a macro before its first screenshot.
- When Roblox's compositor surface changes size, the app rebuilds the frame pool at the actual incoming frame size instead of immediately recreating it from stale window data.
- Concurrent window-geometry changes are retried with freshly read Roblox bounds. A capture that still cannot stabilize ends with a clear user-facing explanation instead of an internal exception type.
- The behavior applies to macro runs, camera setup and Auto Align, placement setup and playback, diagnostic capture, and Deep Debug recording through their shared capture backend.

## Validation

- Cross-thread frame notification, pre-arrived frame consumption, successful surface recovery, and bounded failure regressions pass.
- A live Roblox capture smoke check produced three changing 808 by 611 frames through the updated backend.
- Release solution, complete test suite, repository policy, package verification, and portable-package smoke checks pass before tagging.
