# Expeditions Macro 1.0.10

This patch hardens reward selection and recovery after a reported run missed an Epic reward, incorrectly announced that a rejoin was needed, and then reached a checkpoint without extracting.

## Changes

- Reward detection now combines the stable blue/cyan reward overlay, its segmented progress header, and the visible Select Upgrade controls.
- Purple, gold, and blue reward screens are recognized while one card is still collapsed or moving into place. The macro clicks a complete available card.
- The rigid legacy three-region reward fallback was removed after a complete 11.5-minute match corpus showed that it could score ordinary gameplay near 99%.
- Reward headers explicitly exclude Play-screen recovery, preventing card animations from being mistaken for the Expedition mode tile.
- Lobby, Play, disconnect, and AFK recovery now require consecutive matching captures. A one-frame visual collision no longer abandons the match, resets the observed boss count, or prevents extraction at the configured checkpoint.
- Added seven reviewed 808 by 611 regression captures from the reported failure and a complete manual match: four reward transitions across three rarities and three ordinary-gameplay negatives.
- The complete 2,318-frame manual run was also scanned during release validation to check reward transitions and every recovery decision across the full match timeline.

Existing camera models, placement models, presets, and detector pack 1.0.1 remain compatible. The detector-pack version is unchanged because these fixes are implemented by the app's specialized detectors and state machine.
