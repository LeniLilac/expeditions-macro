# Expeditions Macro v1.3.0-beta.6

This silent beta fixes stale compositor captures, makes scheduled match transitions state-owned, and prevents a correct camera pose from being discarded by non-reversible arrow probes. It does not send a Discord release announcement.

## Capture reliability

- Windows Graphics Capture is continuously drained into the latest FP16 compositor texture, so detectors and deep-debug archives no longer remain frozen on the first captured frame.
- Capture uses the normal hardware Direct3D path on discrete and integrated graphics, with a software WARP fallback when no usable hardware device is available.

## Scheduling and handoffs

- Consecutive Expedition, Story, or Raid tasks using the exact same preset use Repeat Stage instead of reopening the Play interface.
- Different presets and modes leave the current result, party, or map interface through detected live actions and verify the shared Play selector before control returns to the scheduler.
- Challenge victories always return through Play. Challenge defeats use Repeat Stage only while configured retries remain.

## Camera alignment

- A pose that already passes direct goal verification is no longer disturbed by coarse arrow probes.
- Coarse and fine probe round trips are re-observed before their confidence is trusted, preventing variable Roblox key timing from retaining a stale score and forcing another full-turn attempt.
- Full-turn scanning still uses two independently captured frames at every yaw stop; no sampling or precision was removed.

## Validation

- Added camera probe-drift, scheduler continuation, exact-route Repeat Stage, terminal-action, Challenge retry-policy, and live changing-frame capture regressions.
- The release is built only after the complete local Release suite, all checked-in image regressions, checksums, and portable-package verification pass.
