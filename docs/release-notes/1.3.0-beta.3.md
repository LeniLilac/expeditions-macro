# Expeditions Macro v1.3.0-beta.3

This silent beta fixes three issues reported through beta.2 Deep Debug captures and adds a local timeline viewer for those archives. It does not send a Discord release announcement.

## Runtime fixes

- Story recognizes Act, Infinite, and Mastery detail panels across their cyan, green, and purple accents, including both the split Select Stage/Matchmaking layout and the full-width Select Stage layout. The macro clicks the button located in the current frame.
- Saved-team loading recognizes the compact Include Equipment dialog and clicks its detected Include button instead of assuming the older lower position.
- Camera fallback tests a strong saved fine-yaw neighborhood immediately. It no longer completes a full turn before returning to the same candidate, while retaining the configured direct confidence threshold and three-frame final verification.
- A rejected fine-yaw candidate still starts a fresh complete scan without applying the attempt phase twice.

## Deep Debug Viewer

- Added `tools/ExpeditionsMacro.DeepDebugViewer`, a source-only local developer utility for opening a Deep Debug ZIP and replaying captured frames like a video.
- Playback includes scrubbing, stepping, speed control, synchronized nearby detector/workflow/input events, drag-and-drop opening, and friendly corrupt-frame handling.
- A configurable decoded-frame LRU cache defaults to 10 GB, reports live usage, reads ahead during playback, and can be cleared without closing the archive.
- The viewer is intentionally not included in the installer or portable application.

## Faster validation

- The five longest full-corpus golden checks and the remaining golden suite run as six independent jobs alongside fast tests and dark/light UI snapshots.
- Silent beta packaging remains independent, so downloadable artifacts do not wait for the slower validation lanes.

## Validation

- Added reviewed 808 by 611 fixtures for the reported Story and team layouts, including documented unscaled black padding for supplied panel-only Story captures.
- Added focused camera regressions for both successful early fine-neighborhood refinement and false-candidate full-scan fallback.
- The release is built only after the complete local Release suite, UI verification, checksums, and isolated installer/portable smoke checks pass.
