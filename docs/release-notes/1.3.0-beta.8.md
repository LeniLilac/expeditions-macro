# Expeditions Macro v1.3.0-beta.8

This silent beta adds configurable Shift Lock input, repairs Story and Raid launch handling across both party-preview variants, and makes Discord run reports easier to scan. It does not send a Discord release announcement.

## Controls

- Added a dedicated Shift Lock binding under **Settings > Controls**.
- Defaults to Left Ctrl and supports distinct left/right modifier keys, letters, numbers, symbols, numpad keys, function keys, and common control keys through physical scan-code input.

## Navigation

- Story and Raid now launch from both lobby-created and retained post-match party previews.
- Launch verification follows the detected Start action and no longer assumes the post-match Disband action is present.

## Discord reporting

- Story, Raid, Expedition, and Challenge start reports include the current Roblox-window screenshot.
- Victory and Defeat reports show both the completed match runtime and cumulative macro runtime.
- Run facts use Markdown bullets, while start, victory, defeat, and error cards use blue, green, red, and black semantic accents.
- Every app-generated card footer includes the macro version and a Discord-native full timestamp localized for each viewer.

## Validation

- Release solution builds without warnings.
- The complete local Release suite and checked-in image regressions pass.
- Release preflight, repository policy, checksums, native dependency checks, and portable-package smoke verification pass before tagging.
