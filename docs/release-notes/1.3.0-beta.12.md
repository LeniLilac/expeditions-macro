# Expeditions Macro v1.3.0-beta.12

This silent beta restores beta.8 camera-calibration freshness while keeping the Windows 10 capture repair introduced in beta.9, improves long-idle recovery, simplifies Challenge scheduling, and refreshes the preset and connection interfaces. It does not send a Discord release announcement.

## Camera capture and long-idle reliability

- Camera setup, manual Auto Align, and macro alignment discard the queued compositor backlog and wait for a frame produced after a new freshness barrier before scoring movement.
- The scorer no longer accepts either a two-input-old queued pose or the one-input-old pose exposed by the first local beta.11 build.
- Expedition, Challenge, Story, Raid, and Infinite gameplay sends one ordinary `O` pulse every eight minutes. A transient focus or input failure retries after one minute without stopping the task.
- After the AFK Chamber's Return to Lobby action, Story and Raid wait through teleport/loading before testing the configured Play key.

## Challenge scheduling

- Challenge presets no longer contain a separate cooldown Expeditions fallback.
- When the configured Challenge rotation is unavailable, the runner closes its selector, verifies the shared game-mode selector, and returns to the Macro scheduler.
- The scheduler immediately chooses the next highest-priority eligible task, so one priority queue owns every cross-mode decision.
- Existing saved Challenge presets remain readable; the retired cooldown fields are ignored.

## Interface

- Expeditions, Story, and Raid preset editors now use consistent cards for preset selection, route details, models/team, behavior, and advanced tuning.
- Discord notification, failure-ping, and Roblox reconnect inputs use the same compact connection style with inline reveal actions.
- Roblox reconnect includes a **Test link** action that validates and opens the configured private server through Windows without closing an active Roblox process.
- Redundant helper descriptions and idle status placeholders were removed.

## Privacy

- Deep Debug archives redact the active Windows username and profile-directory segment from serialized events, exception details, copied run logs, and copied text model/configuration files.
- Discord webhook values, protected webhook/private-server data, and Discord user IDs remain excluded.

## Validation

- Added post-barrier frame, keepalive timing, AFK handoff, scheduler, reconnect-launch, and archive-level Windows-profile redaction regressions.
- The complete 532-test application/detector suite and all 5 Deep Debug Viewer tests pass.
- Repository policy, release metadata, both-theme UI snapshots, checksums, native dependency verification, and packaged-app smoke tests pass.
