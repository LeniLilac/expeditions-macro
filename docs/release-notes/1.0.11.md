# Expeditions Macro 1.0.11

This patch prevents active Map 2 runs from being abandoned when ordinary gameplay resembles a map-selection screen, hardens computer-vision startup on clean Windows installations, and adds a built-in diagnostic screenshot recorder.

## Changes

- Active matches can enter automatic recovery only after a stable AFK Chamber, Roblox disconnect, or lobby root state is recognized. Play, map selector, and map preview matches cannot initiate recovery by themselves.
- Start, reward, checkpoint, Continue, confirmation, victory, and defeat states take priority over navigation-only visual collisions during gameplay and transition waits.
- Two complete manual runs were scanned during validation. Map 2 reproduced consecutive false map-selector matches during ordinary gameplay; Map 3 contained only legitimate startup navigation detections.
- Added six reviewed 808 by 611 Map 2 and Map 3 gameplay captures that permanently test the recovery boundary.
- Camera calibration no longer exposes an opaque `VisionScorer` type-initializer exception when OpenCV cannot start. The app now reports the underlying native dependency failure with an actionable message.
- Portable and installer builds now include the Microsoft Visual C++ 2015-2022 x64 runtime files required by OpenCvSharp, and release verification checks those dependencies inside the portable ZIP.
- Settings now includes **Debug capture**. Enter a name and interval, arm the workflow, press F6 to begin, and press F6 again to stop. The app records until stopped, restores the original Roblox bounds, and writes PNG frames plus a manifest into a same-name ZIP under the local `diagnostics` folder.

Existing camera models, placement models, presets, and detector pack 1.0.1 remain compatible. The detector-pack version is unchanged because the recovery fix is enforced by the app state machine.

## Downloads

Windows 10 or Windows 11 x64 is required. The binaries are currently unsigned, so Windows may identify the publisher as unknown. Verify the checksums before running a downloaded artifact.
