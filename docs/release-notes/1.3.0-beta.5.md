# Expeditions Macro v1.3.0-beta.5

This silent beta fixes reporting-setting persistence, improves capture consistency on HDR systems, and hardens Story/Raid handoffs between scheduled tasks. It does not send a Discord release announcement.

## Capture and vision

- Detector frames now come directly from the Roblox window compositor surface, so another application covering Roblox does not appear in macro screenshots.
- The capture pipeline keeps HDR and Auto HDR pixels in linear FP16 before converting them to stable SDR RGB for the existing detector logic.
- Story/Raid terminal recognition distinguishes the current compact cyan Victory rail from the visually similar red Defeat panel.

## Scheduling and navigation

- Story and Raid leave their post-match party through the detected Change Gamemode action before the scheduler starts a different mode.
- Story map selectors use their detected Back action when returning to the shared game-mode selector.
- Handoffs fail closed if the expected shared selector cannot be verified, preventing the next task from inheriting a mode-specific panel.

## Settings

- Saving a Macro plan also saves the DPAPI-protected Discord webhook and failure-alert user ID.
- A successful Test webhook action saves the same reporting fields, so reinstalling or replacing the executable no longer appears to reset them.

## Diagnostics

- Added `scripts/New-DiagnosticContactSheet.ps1` to render timestamped frame sequences with detector and automation context from deep-debug archives.
- The repository workflow now asks maintainers to inspect transition contact sheets before isolated frames.

## Validation

- Added restart persistence coverage for reporting settings.
- Added FP16 scRGB conversion, HDR highlight compression, compositor-crop mapping, and overlap-exclusion smoke coverage.
- Added reviewed Story/Raid post-match fixtures and regressions for compact terminal classification and scheduled handoffs.
- The release is built only after the complete local Release suite, all checked-in image regressions, dark/light UI verification, checksums, and installer/portable smoke checks pass.
