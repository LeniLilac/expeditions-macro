# Expeditions Macro 1.0.15

This patch fixes a v1.0.14 run that remained on the **Continue Expedition** confirmation even though the app recognized the dialog at 99% confidence.

## Changes

- The shared Roblox click path now waits long enough for a low-frame-rate client to process the registered cursor move before sending the button press.
- Node confirmation handling now verifies that the modal disappears. If Roblox ignores the first input event, the macro re-focuses Roblox, re-detects the live modal button, and retries up to three times.
- Each retry is tied to a fresh positive confirmation detection. If the modal remains after all three attempts, the macro stops with a specific error instead of waiting indefinitely or clicking an unrelated screen.
- Added a privacy-redacted frame from the supplied 21-frame diagnostic capture. It classifies as `confirm` at 98.6% and maps the modal button to `(340, 340)`, separate from the underlying checkpoint Continue action.

Existing camera models, placement models, presets, and detector pack 1.0.1 remain compatible. The detector-pack version is unchanged because the existing detector already recognizes the screen; this release hardens input acknowledgement and confirmation-state handling.
