# Privacy

Expeditions Macro runs locally on Windows and has no analytics or telemetry.

It reads pixels from the selected Roblox window or screen region and sends simulated input only while a workflow is running. Routine detector and diagnostic frames target the Roblox window's own compositor surface, so another application covering Roblox is not included. HDR/Auto HDR pixels are converted locally to SDR before detection or PNG storage. The explicit camera-region selector can temporarily read the desktop while the user chooses a region. Camera models, placement models, presets, detector packs, settings, logs, and user-requested diagnostic screenshot ZIPs are stored under `%LocalAppData%\ExpeditionsMacro`.

Manual diagnostic capture is started explicitly from Settings. Automatic failure capture is enabled by default and can be disabled there. While a macro runs, it keeps the latest 10 action-state screenshots in memory; after an unexpected Expeditions or Challenge error, it saves those frames and captures 10 more Roblox-client screenshots at 0.5-second intervals. Manual Stop does not trigger it. The app retains the 10 newest automatic error ZIPs and removes older automatic error ZIPs; manual diagnostic archives are not deleted by this cleanup. Diagnostic ZIPs contain captures of the Roblox client plus a local manifest with capture timing, action labels, and app version. Including the current macro run log is also enabled by default and can be disabled independently. The app does not upload these ZIPs. Review them before sharing because Roblox UI and logs can contain account names, chat, local paths, or other personal information.

Deep debug logging is disabled by default and requires an explicit warning confirmation. While enabled, each macro, camera-model, placement-model, or diagnostic-capture operation records every Roblox frame requested by automation, detector state/score events, generated input, observed placement-recording input, and workflow action. It creates a local ZIP after successful, canceled, and failed operations. The ZIP includes sanitized app settings, selected plans and presets, the active detector pack, referenced camera/placement models, and a sanitized run log. Discord webhook values, DPAPI-protected webhook material, and Discord user IDs are omitted. Deep debug frames can still contain any account names, chat, or other content visible inside Roblox. These potentially multi-gigabyte ZIPs remain in `diagnostics/` until the user deletes them and are never uploaded by the app.

Discord reporting is optional. When configured, the app sends only the event summary shown in the UI and a Roblox-window screenshot for victory, defeat, or unexpected recovery. Choosing **Test webhook** sends one Components V2 message without a screenshot or mention. Users may also enter a numeric Discord user ID; an unexpected macro failure then sends five separate Components V2 messages that mention only that ID. Manual Stop does not send these alerts. The webhook value is encrypted for the current Windows user with DPAPI and is never written to the application log. The Discord user ID is stored with the other local app settings.

The app may contact GitHub Releases once per day, or when requested, to check for a newer detector pack. Detector updates are downloaded only after the user accepts the update prompt.
